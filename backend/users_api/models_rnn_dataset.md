# Модель NicknameDataset для RNN датасета

## Общее описание
Модель `NicknameDataset` предназначена для хранения датасета никнеймов, который будет использоваться для обучения RNN (рекуррентных нейронных сетей) для классификации токсичности никнеймов.

## Структура таблицы

### Поля модели

| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| `id` | `BigAutoField` | Первичный ключ | Автоматически генерируется |
| `nickname` | `CharField` | Никнейм для анализа | Уникальный, макс. 50 символов |
| `language_type` | `CharField` | Тип языка никнейма | Выбор из: 'ru' (Русский), 'en' (Английский), 'mixed' (Смешанный) |
| `is_toxic` | `BooleanField` | Метка токсичности | `True` - токсичный (1), `False` - нетоксичный (0) |
| `created_at` | `DateTimeField` | Дата добавления записи | Автоматически устанавливается при создании |

### Выборы (Choices)
```python
LANGUAGE_CHOICES = [
    ('ru', 'Русский'),
    ('en', 'Английский'),
    ('mixed', 'Смешанный'),
]
```

## Индексы для оптимизации запросов

| Индекс | Поля | Назначение |
|--------|------|-----------|
| `idx_nickname` | `nickname` | Ускорение поиска по никнейму |
| `idx_is_toxic` | `is_toxic` | Ускорение фильтрации по токсичности |
| `idx_language_type` | `language_type` | Ускорение фильтрации по языку |
| `idx_created_at` | `created_at` | Ускорение сортировки по дате |

## Метаданные модели

- **Имя модели в админке**: "Никнейм (датасет RNN)"
- **Имя модели во множественном числе**: "Никнеймы (датасет RNN)"
- **Сортировка по умолчанию**: по дате создания (новые сначала)
- **Порядок сортировки**: `['-created_at']`

## Методы модели

### `__str__()`
Возвращает строковое представление объекта в формате:
```
{nickname} ({language_type_display}) - {toxic_status}
```
Пример: `toxic_user123 (Русский) - Токсичный`

### `get_language_type_display()`
Возвращает человекочитаемое название языка (из choices).

## Админка

Модель настроена в админке Django с следующими параметрами:

### Отображение в списке
- Поля: `nickname`, `language_type`, `is_toxic`, `created_at`
- Фильтры: `is_toxic`, `language_type`, `created_at`
- Поиск: по полю `nickname`
- Редактируемое поле: `is_toxic` (можно менять прямо в списке)

### Форма редактирования
Разделена на два блока:
1. **Основная информация**: `nickname`, `language_type`, `is_toxic`
2. **Метаданные** (сворачиваемый блок): `created_at`

### Дополнительные настройки
- Сортировка: по дате создания (новые сначала)
- Иерархия дат: по `created_at` для удобной навигации по датам

## Примеры использования

### Создание записи
```python
nickname = NicknameDataset.objects.create(
    nickname="toxic_user123",
    language_type="ru",
    is_toxic=True
)
```

### Поиск токсичных никнеймов
```python
toxic_nicknames = NicknameDataset.objects.filter(is_toxic=True)
```

### Поиск по языку
```python
russian_nicknames = NicknameDataset.objects.filter(language_type="ru")
```

### Получение новых записей
```python
recent_nicknames = NicknameDataset.objects.all()[:10]  # 10 последних
```

## Назначение для RNN

Эта модель создана специально для:
1. Сбора датасета никнеймов с разметкой токсичности
2. Подготовки данных для обучения RNN
3. Хранения метаданных о никнеймах (язык, дата добавления)
4. Обеспечения быстрого доступа к данным для обучения моделей
